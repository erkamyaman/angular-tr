# Functions should be invoked in event bindings.

Bu tanılama, olay bağlamalarında çağrılmamış fonksiyonları tespit eder.

```angular-ts
import {Component, signal, Signal} from '@angular/core';

@Component({
  template: `<button (click)="(onClick)">Click me</button>`,
})
class MyComponent {
  onClick() {
    console.log('clicked');
  }
}
```

## What's wrong with that?

Olay bağlamalarındaki fonksiyonlar, olay tetiklendiğinde çağrılmalıdır.
Fonksiyon çağrılmazsa, olay tetiklendiğinde çalışmaz.

## What should I do instead?

Olay tetiklendiğinde fonksiyonun çalışması için bir olay bağlamasında kullandığınızda fonksiyonu çağırdığınızdan emin olun.

```angular-ts
import {Component} from '@angular/core';

@Component({
  template: `<button (click)="onClick()">Click me</button>`,
})
class MyComponent {
  onClick() {
    console.log('clicked');
  }
}
```

## Configuration requirements

Herhangi bir genişletilmiş tanılamanın verilmesi için [`strictTemplates`](tools/cli/template-typecheck#strict-mode) etkinleştirilmelidir.
`uninvokedFunctionInEventBinding`, `strictTemplates` dışında ek bir gereksinime sahip değildir.

## What if I can't avoid this?

Bu tanılama, projenin `tsconfig.json` dosyası düzenlenerek devre dışı bırakılabilir:

```json
{
  "angularCompilerOptions": {
    "extendedDiagnostics": {
      "checks": {
        "uninvokedFunctionInEventBinding": "suppress"
      }
    }
  }
}
```

Daha fazla bilgi için [genişletilmiş tanılama yapılandırması](extended-diagnostics#configuration) bölümüne bakın.
