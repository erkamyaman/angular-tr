# Signals must be invoked in template interpolations.

Bu tanılama, şablon interpolasyonlarında çağrılmamış sinyalleri tespit eder.

```angular-ts
import {Component, signal, Signal} from '@angular/core';

@Component({
  template: `<div>{{ mySignal }}</div>`,
})
class MyComponent {
  mySignal: Signal<number> = signal(0);
}
```

## What's wrong with that?

Angular Sinyalleri sıfır argümanlı fonksiyonlardır (`() => T`). Çalıştırıldığında, sinyalin mevcut değerini döndürürler.
Bu, değerlerini oluşturmak için şablon interpolasyonlarında kullanıldığında çağrılmaları gerektiği anlamına gelir.

## What should I do instead?

Değerini oluşturmak için bir şablon interpolasyonu içinde kullandığınızda sinyali çağırdığınızdan emin olun.

```angular-ts
import {Component, signal, Signal} from '@angular/core';

@Component({
  template: `<div>{{ mySignal() }}</div>`,
})
class MyComponent {
  mySignal: Signal<number> = signal(0);
}
```

## Configuration requirements

Herhangi bir genişletilmiş tanılamanın verilmesi için [`strictTemplates`](tools/cli/template-typecheck#strict-mode) etkinleştirilmelidir.
`interpolatedSignalNotInvoked`, `strictTemplates` dışında ek bir gereksinime sahip değildir.

## What if I can't avoid this?

Bu tanılama, projenin `tsconfig.json` dosyası düzenlenerek devre dışı bırakılabilir:

```json
{
  "angularCompilerOptions": {
    "extendedDiagnostics": {
      "checks": {
        "interpolatedSignalNotInvoked": "suppress"
      }
    }
  }
}
```

Daha fazla bilgi için [genişletilmiş tanılama yapılandırması](extended-diagnostics#configuration) bölümüne bakın.
