# Geçersiz Banana-in-Box

Bu tanılama, [iki yönlü bağlamalar](guide/templates/two-way-binding) için ters yazılmış "banana-in-box" sözdizimini tespit eder.

```html
<user-viewer ([user])="loggedInUser" />
```

## Bunda ne yanlış?

Mevcut haliyle, `([var])` aslında `[var]` adında bir [olay bağlamasıdır](guide/templates/event-listeners).
Yazar muhtemelen bunu `var` adlı bir değişkene iki yönlü bağlama olarak amaçlamıştır, ancak yazıldığı şekliyle bu bağlamanın hiçbir etkisi yoktur.

## Bunun yerine ne yapmalıyım?

Yazım hatasını düzeltin.
Adın da önerdiği gibi, muz `(` kutunun `[]` _içine_ gitmelidir.
Bu durumda:

```html
<user-viewer [(user)]="loggedInUser" />
```

## Yapılandırma gereksinimleri

Herhangi bir genişletilmiş tanılamanın verilmesi için [`strictTemplates`](tools/cli/template-typecheck#katı-mod) etkinleştirilmelidir.
`invalidBananaInBox`, `strictTemplates` dışında ek bir gereksinime sahip değildir.

## Bunu önleyemezsem ne olur?

Bu tanılama, projenin `tsconfig.json` dosyası düzenlenerek devre dışı bırakılabilir:

```json
{
  "angularCompilerOptions": {
    "extendedDiagnostics": {
      "checks": {
        "invalidBananaInBox": "suppress"
      }
    }
  }
}
```

Daha fazla bilgi için [genişletilmiş tanılama yapılandırması](extended-diagnostics#yapılandırma) bölümüne bakın.
