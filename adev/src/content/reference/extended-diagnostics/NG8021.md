# Defer Trigger Misconfiguration

Bu tanılama, `@defer` bloklarındaki erişilemez veya gereksiz tetikleyicileri tespit eder.

```typescript
import {Component} from '@angular/core';

@Component({
  template: `
    @defer (on immediate; on timer(500ms)) {
      <large-component />
    }
  `,
})
class MyComponent {}
```

## What's wrong with that?

Bu tanılama, defer tetikleyici yapılandırmasında şunlara yol açan birkaç sorunlu kalıbı tanımlar:

- **Gereksiz kod** - davranışı hiçbir zaman etkilemeyen kod
- **Kaçırılan optimizasyon fırsatları** - daha iyi performans için
- **Erişilemez prefetch tetikleyicileri** - hiçbir zaman çalışmayacak olan tetikleyiciler

## Diagnostic warning cases

Bu tanılama aşağıdaki sorunlu kalıpları işaretler:

### `immediate` with prefetch triggers

**Kötü -- prefetch hiçbir zaman çalışmaz**

```typescript
@Component({
  template: `
    @defer (on immediate; prefetch on idle) {
      <my-cmp />
    }
  `,
})
class MyComponent {}
```

**İyi -- gereksiz prefetch'i kaldırın**

```typescript
@Component({
  template: `
    @defer (on immediate) {
      <my-cmp />
    }
  `,
})
class MyComponent {}
```

### Prefetch timer not earlier than main timer

**Kötü -- prefetch ana tetikleyiciden daha geç**

```typescript
@Component({
  template: `
    @defer (on timer(100ms); prefetch on timer(3000ms)) {
      <my-cmp />
    }
  `,
})
class MyComponent {}
```

**Kötü -- eşit zamanlama herhangi bir fayda sağlamaz**

```typescript
@Component({
  template: `
    @defer (on timer(500ms); prefetch on timer(500ms)) {
      <my-cmp />
    }
  `,
})
class MyComponent {}
```

**İyi -- prefetch daha erken tetiklenir**

```typescript
@Component({
  template: `
    @defer (on timer(1000ms); prefetch on timer(500ms)) {
      <large-component />
    }
  `,
})
class MyComponent {}
```

### Identical prefetch and main triggers

**Kötü -- aynı viewport tetikleyicisi**

```typescript
@Component({
  template: `
    @defer (on viewport; prefetch on viewport) {
      <my-cmp />
    }
  `,
})
class MyComponent {}
```

**Kötü -- aynı etkileşim hedefi**

```typescript
@Component({
  template: `
    <button #loadBtn>Load</button>
    @defer (on interaction(loadBtn); prefetch on interaction(loadBtn)) {
      <large-component />
    }
  `,
})
class MyComponent {}
```

**İyi -- gereksiz prefetch'i kaldırın**

```typescript
@Component({
  template: `
    <button #loadBtn>Load</button>
    @defer (on interaction(loadBtn)) {
      <large-component />
    }
  `,
})
class MyComponent {}
```

**İyi -- prefetch ve ana tetikleyici için farklı hedefler kullanın**

```typescript
@Component({
  template: `
    <div #hoverArea>Hover to prefetch</div>
    <button #clickBtn>Click to load</button>
    @defer (on interaction(clickBtn); prefetch on hover(hoverArea)) {
      <large-component />
    }
  `,
})
class MyComponent {}
```

## Configuration requirements

Herhangi bir genişletilmiş tanılamanın verilmesi için [`strictTemplates`](tools/cli/template-typecheck#strict-mode) etkinleştirilmelidir.
`deferTriggerMisconfiguration`, `strictTemplates` dışında ek bir gereksinime sahip değildir.

## What if I can't avoid this?

Bu tanılama, projenin `tsconfig.json` dosyası düzenlenerek devre dışı bırakılabilir:

```json
{
  "angularCompilerOptions": {
    "extendedDiagnostics": {
      "checks": {
        "deferTriggerMisconfiguration": "suppress"
      }
    }
  }
}
```

Daha fazla bilgi için [genişletilmiş tanılama yapılandırması](extended-diagnostics#configuration) bölümüne bakın.
